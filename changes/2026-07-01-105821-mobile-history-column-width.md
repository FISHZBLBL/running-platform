# 2026-07-01 10:58:21 手机端历史记录列宽优化

## 做了哪些改动

- 手机端历史记录的跑步列和体重列改为按内容宽度排列。
- 跑步卡片不再被固定列宽强行拉伸。
- 跑步和体重两个区域仍保持横向并排，可通过横向滑动查看。

## 为什么做这个改动

- 上一版为了让跑步和体重在手机端并排，给跑步列设置了较大的最小宽度。
- 当单条跑步记录内容本身不宽时，卡片右侧会出现明显空白，视觉上浪费空间。

## 解决了什么问题

- 减少手机端历史记录跑步卡片右侧的大面积空白。
- 保留跑步和体重并排管理的便利性。
- 让手机端历史记录列表更紧凑、更适合编辑操作。

## 注意事项

- 这次只修改本地代码，未部署到 Netlify。
