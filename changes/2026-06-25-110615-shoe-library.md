# 2026-06-25 11:06:15 +08:00 - 鞋库与跑步记录选鞋

## 改动内容
- 新增跑鞋数据结构 `RunningShoe`，跑步记录新增 `shoeId` 字段，用于关联每次跑步使用的跑鞋。
- 新增 COS key 规则：
  - `users/{username}/shoes/{shoeId}.json`
  - `users/{username}/shoes/{shoeId}/photos/{fileId}.{ext}`
- 新增 Netlify API：
  - `GET /api/shoes`
  - `POST /api/shoes`
  - `PUT /api/shoes/:id`
  - `DELETE /api/shoes/:id`
  - `POST /api/shoe-photo`
- 本地局域网预览模拟后端同步支持鞋库、跑鞋照片、跑鞋删除和跑步记录选鞋。
- 顶部标题下方新增页面切换按钮：`主页` 和 `鞋库`。
- 新增鞋库界面，可添加跑鞋名称、预计寿命里程和跑鞋照片。
- 鞋库卡片显示跑鞋照片、名称、累计使用里程和寿命进度条。
- 跑步记录表单新增“本次跑鞋”选择框。
- 历史记录中的跑步条目会显示关联跑鞋名称。

## 为什么做这个改动
- 用户需要记录每次跑步使用的跑鞋，并在单独界面管理跑鞋。
- 跑鞋寿命里程可以帮助后续观察不同跑鞋和跑步表现之间的关系。
- 顶部加入主页/鞋库切换，避免所有功能继续堆在一个页面里。

## 解决的问题
- 解决了跑步记录无法关联跑鞋的问题。
- 解决了无法统计每双鞋累计跑量的问题。
- 为后续扩展“跑鞋型号、寿命提醒、不同跑鞋表现对比”打基础。

## 说明
- 当前进度条只展示累计里程占预计寿命的比例，不会限制继续选择这双鞋。
- 跑鞋照片第一版支持用户自行上传。
- 联网搜索匹配跑鞋图片暂未实现；后续更适合做成“搜索候选图 -> 用户确认”的流程，避免同名鞋款和配色误匹配。
- 如果 COS 桶允许通过 `COS_DOMAIN` 公网读取，正式环境会直接显示上传后的跑鞋照片；如果未来改成私有桶，需要再加服务端图片代理或临时签名 URL。
