# 2026-06-25 15:23:33 +08:00 - 新增记录页签

## 改动内容
- 顶部页面切换从 `主页 / 鞋库` 调整为：
  - `主页`
  - `记录`
  - `鞋库`
- 新增 `记录` 页面，用于集中管理：
  - 跑步记录
  - 体重记录
  - 历史记录
- `主页` 现在只保留分析展示内容：
  - 趋势图
  - 目标预测
  - 概览指标
- 历史记录从主页移到记录页，并作为记录页常驻模块展示。
- 历史记录内部的月份仍默认收起，避免进入记录页后内容过长。
- 从历史记录点击编辑跑步或体重时，会切换/定位到记录页顶部，方便直接修改表单。

## 为什么做这个改动
- 跑步录入、体重录入和历史管理都属于“数据记录与管理”工作流，放在一个页面里更清晰。
- 主页更适合作为分析页，减少录入表单对图表和预测内容的干扰。
- 顶部将 `记录` 放在 `主页` 和 `鞋库` 中间，符合从查看分析到管理记录再到管理装备的导航顺序。

## 解决的问题
- 解决主页内容过多、分析展示和数据录入混在一起的问题。
- 让用户可以在独立的记录页完成新增、编辑和历史管理。
- 保持鞋库作为独立装备管理页。
