# 历史记录删除功能

- 日期时间：2026-06-25 10:31:57 +08:00
- 分支：`codex/dev-preview`
- 状态：本地预览，未推送、未部署

## 做了哪些改动

1. 在历史记录管理中为跑步记录增加“删除”按钮。
2. 在历史记录管理中为体重记录增加“删除”按钮。
3. 删除前增加浏览器确认提示，降低误删风险。
4. 本地预览 API 增加跑步记录和体重记录删除能力。
5. Netlify Functions 增加体重记录删除能力，跑步记录删除沿用已有接口。
6. 编辑体重记录时，如果日期被修改，保存新日期后会自动删除旧日期记录，避免留下错误日期数据。
7. 删除按钮增加危险操作样式，与编辑按钮区分。

## 为什么做这些改动

- 之前历史数据只能编辑，无法彻底移除录错的记录。
- 体重记录以日期作为唯一键，日期录错后仅编辑为新日期会产生一条新记录，旧错误日期仍然存在。
- 跑步和体重历史数据会参与趋势图和预测，错误数据如果无法删除会污染趋势判断。

## 解决的问题

- 用户可以直接清理录错的跑步或体重记录。
- 体重日期录错后，可以通过编辑修正，系统自动移除旧日期记录。
- 历史记录管理更完整，后续预测和图表的数据质量更可控。

## 验证计划

- 运行 `npm run test`。
- 运行 `npm run build`。
- 在本地预览中删除一条体重记录，确认历史列表和趋势数据刷新。
- 编辑体重记录日期，确认旧日期记录不再保留。
