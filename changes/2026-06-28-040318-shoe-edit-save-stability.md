# 2026-06-28 04:03:18 跑鞋编辑保存稳定性修复

## 做了哪些改动

- 跑鞋编辑接口 `PUT /api/shoes/:id` 不再先读取 COS 中的旧跑鞋记录。
- 编辑保存时直接使用前端提交的完整跑鞋数据写回 COS，减少一次云存储请求。
- COS 网络层增加更明确的错误提示，将底层 `fetch failed` 转换为“服务器连接腾讯 COS 失败：...”。

## 为什么做这个改动

- 编辑跑鞋名称时，前端已经携带了原跑鞋的 `photoKey`、`photoUrl` 和 `createdAt`，后端不需要为了保留这些字段再额外读一次 COS。
- 原流程需要先 GET 再 PUT，任意一次 COS 网络波动都会导致保存失败。
- `fetch failed` 过于模糊，无法判断是浏览器请求失败、Netlify Function 失败，还是腾讯 COS 连接失败。

## 解决了什么问题

- 降低跑鞋名称编辑保存时出现 `fetch failed` 的概率。
- 减少一次 COS 请求，编辑保存更快、更稳。
- 后续如果 COS 连接异常，页面能显示更接近真实原因的错误信息。

## 注意事项

- 这个改动不会影响新增跑鞋、删除跑鞋或跑步记录选择跑鞋。
- 编辑接口仍然需要成功写入 COS；如果腾讯 COS 或密钥配置异常，保存仍会失败，但错误信息会更清楚。
