# 2026-06-28 05:19:34 历史记录缺失数据状态标记

## 做了哪些改动

- 历史跑步记录中固定显示两个状态徽标：分段数据、跑鞋数据。
- 有对应数据时显示彩色图标和勾选。
- 没有对应数据时显示红色边框和叉。
- 将跑鞋徽标从抽象字符改成内联 SVG 跑鞋图标。

## 为什么做这个改动

- 只显示“有”的状态时，用户无法快速区分“没有录入”和“只是没显示”。
- 原跑鞋字符图标不够像跑鞋，识别成本高。
- 固定两个状态位能让每条记录的完整程度更容易扫描。

## 解决了什么问题

- 解决没有分段或没有跑鞋时缺少明确提示的问题。
- 解决跑鞋图标语义不清的问题。
- 保持界面没有直接写“有分段记录 / 有跑鞋记录”这类长文本。

## 注意事项

- 这次只修改本地代码，没有部署到 Netlify。
- 徽标的 `title` 和 `aria-label` 仍保留状态信息，方便鼠标悬停和辅助技术识别。
