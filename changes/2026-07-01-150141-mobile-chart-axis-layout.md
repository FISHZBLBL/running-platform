# 2026-07-01 15:01:41 手机端图表坐标轴布局优化

## 改动内容
- 为首页独立图表增加窄屏视口判断。
- 在手机端将“跑步表现趋势”的坐标轴改为紧凑布局：
  - 隐藏手机端 Y 轴标题，减少文字占位。
  - 减少左右 grid 留白。
  - 保留距离轴刻度，隐藏心率轴刻度，将心率主要通过曲线和 tooltip 查看。
  - 图例改为可滚动图例，避免挤压绘图区。
- 在手机端同步优化“体重与跑步表现”散点图：
  - 减少左右坐标轴占位。
  - 隐藏 Y 轴标题，保留数据点和 tooltip。
- 为窄屏判断补充旧版 Safari 的 `matchMedia.addListener` fallback。

## 为什么做这个改动
- 手机竖屏宽度有限，第一张图同时展示配速、距离、心率三组 Y 轴时，右侧两个轴会占用大量宽度，导致真正的数据绘图区被压得很窄。
- 体重关系图也有双 Y 轴，同样会在手机上被坐标轴文字挤压。

## 解决的问题
- 手机端首页前两张图不再被多条 Y 轴压缩成很窄的图。
- 保留关键数据展示，同时把部分详细读数交给 tooltip，提升手机竖屏下的可读性。
- 降低不同手机浏览器对窄屏布局识别不一致的风险。
