# 2026-06-28 03:43:51 跑鞋照片代理读取与编辑功能

## 做了哪些改动

- 将正式环境跑鞋照片显示从 COS 公开直链改为同源接口 `/api/shoe-photo?key=...` 代理读取。
- 为存储适配器新增 `getFile` 能力，Netlify Function 可以从 COS 读取图片二进制并返回给浏览器。
- 跑鞋照片上传后返回同源图片地址，避免浏览器直接访问私有 COS 对象。
- 鞋库新增编辑模式，可以修改跑鞋名称，也可以重新选择照片替换原照片。
- 跑鞋卡片新增“编辑”按钮，并调整编辑/删除按钮布局。

## 为什么做这个改动

- 本地预览的跑鞋照片是压缩后的 data URL，保存在浏览器 localStorage 中，所以可以直接显示。
- 正式环境的跑鞋照片保存在腾讯 COS，如果 COS 桶没有开放公共读取权限，之前保存的公开 URL 会被浏览器拒绝访问。
- 通过 Netlify Function 代理读取，可以让 COS 保持私有，浏览器只访问本站同源接口，安全性和可用性都更好。

## 解决了什么问题

- 解决部署后鞋库照片无法正常显示的问题。
- 避免为了展示照片而把整个 COS 存储桶改成公开读。
- 支持跑鞋信息录入错误后的后续修改，减少只能删除重建的操作成本。

## 注意事项

- 已有跑鞋记录只要保存了 `photoKey`，前端会优先用新的代理地址显示。
- 如果历史记录里只有旧的 `photoUrl`、没有 `photoKey`，仍会回退使用旧 URL。
- 替换照片不会自动删除旧 COS 图片，后续如果需要节省存储空间，可以再做旧照片清理逻辑。
