# 2026-07-01 18:17:47 手机日期输入和图例分页修复

## 改动内容
- 给表单 `label`、`input`、`select`、`textarea` 增加 `min-width: 0` 和 `max-width: 100%` 约束。
- 在手机端专门处理 `date` 和 `datetime-local` 输入框：
  - 去除移动浏览器原生控件的额外 appearance。
  - 限制最大宽度，防止撑出表单卡片。
  - 将日期文本改为左对齐，减少视觉溢出感。
- 将“跑步表现趋势”手机端图例从分页滚动图例改回普通图例。
- 缩小手机端图例的字体、图标和间距，并给图表顶部增加空间，允许图例自然换行显示。

## 为什么做这个改动
- 手机浏览器，尤其 iOS Safari/Chrome，会给日期输入框设置原生最小宽度，导致日期选择框在窄屏下超出卡片。
- 上次为了避免图例挤压图表，启用了 ECharts 的 scroll legend，但它会显示 `1/2` 分页，不符合当前页面的阅读习惯。

## 解决的问题
- 跑步记录和体重记录里的日期选择框不再横向超出窗口。
- 首页第一张图的图例不再被拆成两页。
- 移动端图例可以完整展示，同时不明显压缩绘图区。
